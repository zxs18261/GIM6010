# CAN 通信协议定义

经典 CAN 2.0A（标准帧，DLC=8 除注明外），默认 1Mbps（可配 1M/500k/250k/125k），默认节点 ID = 1（范围 1~63）。硬件滤波仅放行三个 ID：`NodeId`、`0x000`（广播）、`0x600+NodeId`（配置请求）。实现见 [app_can_protocol.c](../Application/app/app_can_protocol.c)。

## 1. MIT 运控通道（CAN ID = NodeId）

与 SteadyWin GIM 家族 / MIT mini-cheetah 协议同构，量程定义在 [motor_config.h](../Application/config/motor_config.h)：

| 量 | 范围 | 位宽 | 量纲（输出轴） |
|---|---|---|---|
| P | ±12.5 | 16 | rad |
| V | ±65 | 12 | rad/s |
| KP | 0~500 | 12 | N·m/rad |
| KD | 0~5 | 12 | N·m·s/rad |
| T | ±18 | 12 | N·m |

编码：`uint = (x - min) × ((1<<bits)-1) / (max - min)`，高位先行。

### 1.1 特殊帧（8 字节，前 7 字节 0xFF）

| data[7] | 动作 |
|---|---|
| `0xFC` | 进入电机模式：切 MIT 模式并使能（需已校准，目标初始化为当前位置、KP/KD/Tff=0，无跳动） |
| `0xFD` | 退出电机模式：封波 |
| `0xFE` | 设当前位置为零位 |

广播 ID `0x000` 同样接受特殊帧（多电机同步使能/失能），但**不回发反馈**（避免多节点同时以 ID 0x000 应答产生仲裁冲突）；广播收到的常规命令帧一律忽略。

### 1.2 命令帧（仅 MIT 模式响应）

```
B0 = P[15:8]        B1 = P[7:0]
B2 = V[11:4]        B3 = V[3:0]<<4 | KP[11:8]
B4 = KP[7:0]        B5 = KD[11:4]
B6 = KD[3:0]<<4 | T[11:8]           B7 = T[7:0]
```

控制律（20kHz 执行）：`τ = KP·(P−p) + KD·(V−v) + T`，`iq = τ / (Kt_rotor×减速比)`，受电流限幅（含温度降额）约束。每条有效命令帧喂指令看门狗——凡**由 CAN 发起使能**的闭环模式（MIT 及配置通道 0x02 使能的转矩/转速/位置模式）超时即封波并锁存 `FAULT_CAN_TIMEOUT`（默认 500ms，参数 `timeout` 可调/0 关闭）；shell 调试发起的使能不受此约束。

### 1.3 反馈帧（DLC=6，发往 ID `0x000`）

每收到一条特殊帧或有效命令帧回发一条：

```
B0 = NodeId
B1 = P[15:8]   B2 = P[7:0]
B3 = V[11:4]   B4 = V[3:0]<<4 | T[11:8]   B5 = T[7:0]
```

T 为估计输出转矩 = iq 实测 × Kt_out，量程同上表反解。

## 2. 配置通道（请求 ID = 0x600+NodeId，应答 ID = 0x680+NodeId）

请求 `data[0]` = 命令码；浮点均为小端 float32 放 `data[4..7]`。通用应答格式：`B0=命令码, B1=状态(1 成功/0 拒绝), B4..7=float 值`。

| 命令码 | 功能 | 请求参数 | 说明 |
|---|---|---|---|
| 0x01 | 读状态 | — | 应答：B1=模式, B2=故障码, B3=bit0 使能/bit1 已校准, B4..5=位置×100(int16 BE), B6..7=速度×100 |
| 0x02 | 使能 | B1=模式枚举(1 开环/2 转矩/3 转速/4 位置/5 MIT) | 封波状态下切模式并使能 |
| 0x03 | 失能 | — | |
| 0x04 | 清故障 | — | |
| 0x05 | 启动预定位校准 | — | 电机会旋转！ |
| 0x06 | 设零位 | — | |
| 0x07 | 目标值 | float | 按当前模式解释：转矩[N·m]/转速[rad/s]/位置[rad]；喂看门狗 |
| 0x10 | 读参数 | B1=索引 | 应答 float |
| 0x11 | 写参数 | B1=索引 + float | 只读项/越界拒绝 |
| 0x12 | 保存参数 | — | 后台封波时落盘 |
| 0x13 | 恢复默认 | — | 仅封波状态，不落盘；应答以旧 ID 发出后按默认 NodeId/波特率/终端重配 CAN |

参数索引（与 shell 参数名一一对应）：

| 索引 | 名称 | shell 名 | 属性 |
|---|---|---|---|
| 0 | 节点 ID (1~63) | id | 读写，写后 CAN 重配 |
| 1 | 波特率枚举 (0=1M,1=500k,2=250k,3=125k) | baud | 读写，写后 CAN 重配 |
| 2 | 120Ω 终端 (0/1) | term | 读写，即时生效 |
| 3 | 指令超时 [ms]（0 关闭） | timeout | 读写 |
| 4/5 | 速度环 Kp/Ki | kp/ki | 读写，即时生效 |
| 6 | 位置环 Kp | poskp | 读写 |
| 7 | 电流限幅 [A] | curmax | 读写（≤电机峰值） |
| 8 | 速度限幅 [rad/s] | spdmax | 读写 |
| 9 | LED 亮度 (0~255) | bright | 读写 |
| 10 | 电角偏置 [rad] | offset | 只读（校准写入） |
| 11 | 极对数 | pp | 只读（校准实测） |
| 12 | 编码器方向反转 | encinv | 只读（校准写入） |
| 13 | 已校准标志 | caldone | 只读 |

## 3. 多机组网

- 各节点经 XT30(2+2) 菊花链共享电源与 CAN；链路首末两个节点 `set term 1`，中间节点 `set term 0`（`save` 持久化）。
- 节点 ID 分配：逐台上电经配置通道 0x11/索引 0 写入后 `save`。
- 注意：MCU 复位到 GPIO 初始化期间 EN_120 处于接入状态（PA15 复位态上拉），对总线的扰动为毫秒级瞬态。
